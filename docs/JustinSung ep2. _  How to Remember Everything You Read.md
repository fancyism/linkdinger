Procederal  
ข้อมูลแบบ “วิธีทำ/ขั้นตอน” เช่น หนังสือวิธีการฟังเสียงหัวใจ วิธีวัดความดัน เป็นต้น หรือจะสาย dev การเขียนโค้ด syntax วิธีใช้ library ฯลฯ เป็นข้อมูลแนว “How to” – Coding / Languages จะชัดมากสำหรับหมวดหมู่นี้   
KEY : \[ Practice make perfect \] \* \[ Consuming \+ Digesting \] \=\> quick practice  
Process  : ทันทีที่รับข้อมูล Procederal เข้ามา ให้ฝึกใช้จริง ให้เร็วๆ ที่สุดในชีวิตจริง “แทนการนั่งอ่าน” เขียนโน็ตยาวๆ –ถ้ายังไม่มีเวลาฝึก\! ให้หยุดอ่านเรื่องนั้นแล้วไปทำอย่างอื่น หรือ รอจนกว่าจะมีเวลาฝึก อย่าฝืนอ่านต่อเพื่อ “รูัสึกว่าทำอะไรอยู่เพื่อความสบายใจ” เพราะจะลืม 90% อยู่ดี    
Example  : 

1. หนังสือสอน “ท่าตรวจร่างกาย” ในแพทย์ – แทนที่จะอ่านทั้งบทแล้วค่อยไปฝึก เขาแนะนำให้อ่าน step เล็กๆ แล้วไปลองทำทันที เช่น ดูขั้นตอนการฟังหัวใจ แล้วไปฝึกฟังจริง ไม่ใช่เก็บไว้ฝึกทีเดียวอีกอาทิตย์  
2. อ่านวิธีใช้ Redis หรือ Modal แค่พอเข้าใจ flow จากนั้นเปิดโปรเจกต์เล็กๆ แล้วลอง implement ตามทันที เช่น ทำ endpoint ที่ยิงไป cache ผลลัพธ์จริง ไม่ใช่อ่านจนจบก่อนค่อยลอง

**Analogous \- Critique**   
ข้อมูลใหม่ที่ “เชื่อมโยง/คล้าย” สิ่งที่เรารู้มาก่อน เช่น เรียน physiology แล้วนึกถึงท่าว่ายน้ำที่เคยฝึก หรือปัญหาใหม่ที่คล้ายโจทย์เก่า

“Analogous information is the information that is related  
to something you already have prior knowledge about.”

Key : Analogous Information   
Process :  สมมุติเราเป็นนักว่ายน้ำโอลิมปิก และเรากำลังเรียนรู้เกี่ยวกับ Physiology ของการหดตัวของกล้ามเนื้อ เมื่อเราดูวงจรหดตัวของกล้ามเนื้อ นั้นทำให้เรานึกถึงเทคนิคการว่ายน้ำซึ่งจะเชื่อมกับข้อมูลใหม่เรื่อง การหดตัวของกล้ามเนื้อในวิชา physiology แล้วทำการเปรียบเทียบขึ้นมา 

- STEP CRITIQUE เมื่อเราเห็นว่าว่าอะไร “คล้ายของเดิม” ให้เราทำ 4 อย่าง   
  - ถามว่า : เหมือนกันตรงไหนบ้างแบบเฉพาะเจาะจง เช่น ว่ายน้ำกับหดตัวกล้ามเนื้อมีความคล้าย เกี่ยวข้อง ด้านใดบ้าง  
  - ถามว่า : ต่างกันตรงไหนบ้าง  
  - ถามว่า : ในสถานการณณ์ไหน analogy นี้ ใช้ไม่ได้   
  - ถามว่า : มี analogy ที่ดีกว่านี้ไหม หรือจะปรับยังไงให้แม่นยำมากขึ้น

ด้วย 4 STEP นี้จะช่วยทำให้เราเข้าใจได้ลึกซึ้งและจดจำมากขึ้นสเมือนค่อยๆ ย้อนอดึตของเราที่ลึกซึ้งมากขึ้นมาเปรียบเทียบกับสิ่งที่เป็นเนื้อหาใหม่ของเรา   
\*\*ใช้ analogy เข้าไปในส่วนของกระบวนการ Procederal ได้ด้วยเพื่อใช้ในการทำความเข้าใจให้เข้าใจง่ายขึ้น และอธิบายให้ตัวเองเข้าใจมากขึ้น เป็น subset ของ Analogous ไหม
