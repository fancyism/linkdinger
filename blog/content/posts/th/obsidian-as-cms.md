# ปัญหาของ CMS แบบดั้งเดิม

WordPress, Ghost, Contentful—ทั้งหมดนี้เป็นเครื่องมือที่ทรงพลัง แต่ก็เพิ่มความยุ่งยากอย่างมาก เมื่อมีไอเดียผุดขึ้นมา คุณต้องเข้าสู่ระบบพอร์ทัลเว็บ จัดการกับตัวแก้ไข WYSIWYG ที่มักจะทำให้การจัดรูปแบบผิดเพี้ยน และกังวลเกี่ยวกับการสำรองข้อมูลฐานข้อมูล

## เข้าสู่ Local Vault

ด้วยการใช้ **Obsidian** เป็นแหล่งข้อมูลเดียวที่เชื่อถือได้ ฉันจึงกลับมาเป็นเจ้าของข้อมูลของฉันได้อย่างสมบูรณ์ มันเป็นเพียงโฟลเดอร์ของไฟล์ข้อความเท่านั้น

### เวิร์กโฟลว์

1. **เขียนในเครื่อง**: ไม่ต้องใช้อินเทอร์เน็ต ค้นหาได้ทันที และเชื่อมโยงสองทาง
2. **เพิ่ม Frontmatter**: เพียงแค่ติดแท็กบันทึกด้วย `publish: true` ก็จะทำเครื่องหมายให้สาธารณะ
3. **The Watcher**: สคริปต์ Python จะตรวจสอบ vault หากพบรูปภาพใหม่ จะบีบอัด อัปโหลดไปยัง Cloudflare R2 และเขียนลิงก์ markdown ใหม่
4. **Git Sync**: สคริปต์จะคอมมิตและพุชการเปลี่ยนแปลงอย่างเงียบๆ Vercel จะรับไปและปรับใช้ส่วนหน้าของ Next.js ใหม่

ไม่มีฐานข้อมูล ไม่มีการเข้าสู่ระบบ เพียงแค่การเขียนล้วนๆ นี่คือปรัชญาที่แท้จริงเบื้องหลังโครงสร้างพื้นฐานของ Linkdinger
