# วิสัยทัศน์

จะเป็นอย่างไรถ้าบล็อกส่วนตัวของคุณดูเหมือนได้รับการออกแบบโดยคณะกรรมการ Awwwards โดยใช้เพียงแค่ **Obsidian** และ **Git**?

นั่นคือ Linkdinger: บล็อกอัจฉริยะที่การเขียนเกิดขึ้นใน Obsidian รูปภาพจะถูกอัปโหลดไปยัง Cloudflare R2 โดยอัตโนมัติ และทุกอย่างจะซิงค์กับ GitHub — ทั้งหมดนี้ผ่านกระบวนการ daemon เดียว

## Dark Glassmorphism: มากกว่าแค่การเบลอ

Glassmorphism เป็นที่นิยมมาระยะหนึ่งแล้ว แต่การนำไปใช้ส่วนใหญ่พลาดประเด็นสำคัญ มันไม่ใช่แค่การเพิ่ม `backdrop-filter: blur(20px)` ให้กับทุกสิ่ง — แต่มันคือการ **จำลองฟิสิกส์ของกระจกจริง**

### สามเสาหลัก

1.  **ความโปร่งใสหลายชั้น** — การไล่ระดับสีแบบ Alpha-channel ไม่ใช่ความทึบแบบเรียบ
2.  **การเบลอเชิงแสง** — จุดที่เหมาะสมที่สุด 10-20px เพียงพอที่จะบิดเบือนแต่ไม่ลบเลือน
3.  **ขอบดักจับแสง** — 1px `rgba(255,255,255,0.1)` เลียนแบบการหักเหของขอบ

```css
.glass-card {
  background: rgba(255, 255, 255, 0.03);
  backdrop-filter: blur(20px);
  border: 1px solid rgba(255, 255, 255, 0.08);
  box-shadow: inset 0 1px 0 rgba(255, 255, 255, 0.05);
}
```

### ส่วนสำคัญที่ขาดหายไป

นี่คือสิ่งที่นักพัฒนา 90% เข้าใจผิด: **กระจกจะมองไม่เห็นบนพื้นหลังสีเข้มทึบ** คุณต้องมีลูกแก้วไล่ระดับสีรอบข้าง — ก้อนสีสดใส (สีม่วงเข้ม, สีน้ำเงินไฟฟ้า, สีชมพูร้อน) ลอยอยู่ด้านหลัง UI ของคุณ หากไม่มีสิ่งเหล่านี้ คุณก็จะได้แค่กล่องสีเทา

## เทคโนโลยีที่ใช้

| ส่วนประกอบ | เทคโนโลยี |

## สถาปัตยกรรมสองธีม

การรองรับทั้งธีมมืดและธีมสว่างด้วย glassmorphism นั้นซับซ้อน ประเด็นสำคัญคือ: **กลับสูตรของกระจก**

-   **โหมดมืด**: `bg-white/[0.03]` + `border-white/[0.08]`
-   **โหมดสว่าง**: `bg-black/[0.03]` + `border-black/[0.08]`

การไล่ระดับสีรอบข้างก็เปลี่ยนไปเช่นกัน — จากลูกแก้วนีออนเป็นสีพาสเทลอ่อนๆ

> UI ที่ดีที่สุดคือ UI ที่ให้ความรู้สึกมีชีวิตชีวา พื้นผิวกระจกจะตอบสนองต่อสีที่อยู่ด้านหลัง สร้างประสบการณ์แบบไดนามิกที่การออกแบบแบบเรียบๆ ไม่สามารถเทียบได้
